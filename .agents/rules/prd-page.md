# 루미너스의원 사용자 페이지 기능 명세서 (User-facing PRD)

본 문서는 루미너스의원 웹사이트의 **일반 사용자 대상** 페이지 및 기능 명세서입니다.
관리자 페이지(`/admin/*`)에 대한 명세는 `prd-admin.md`를 참조하세요.

> **디자인 레퍼런스:** `html_resource/` 폴더에 이미 완성된 HTML 디자인 파일이 있습니다. 각 페이지 구현 시 해당 HTML 파일의 레이아웃, 스타일, 애니메이션을 그대로 재현해야 합니다.

---

## 1. 상세 페이지별 요구사항

### 1.1 루미너스의원 (Clinic Info)
- **의료진 소개** (`html_resource/doctors.html`): 의료진 프로필 사진(R2)과 약력(D1)을 동적으로 매핑하여 노출합니다.
- **둘러보기** (`html_resource/tour.html`): 병원 내부 인테리어 고화질 이미지를 갤러리 형태로 구성합니다.
- **오시는 길** (`html_resource/location.html`): 지도 API를 연동하고 본사 위치 및 교통편 정보를 제공합니다.
- **루미너스 장비:** 텍스트 설명 대신 장비 상세 정보가 포함된 이미지로 대체하여 장비의 하이엔드 퀄리티를 직관적으로 전달합니다.
- **전후사진** (`html_resource/results.html`): 시술 전/후 비교 이미지를 노출하며, 의료법 준수를 위해 비로그인 시 썸네일 블러 처리 로직을 적용합니다.

### 1.2 리프팅 센터 (Lifting Center)
- **목록 페이지** (`html_resource/equipment.html`): 루미너스의원이 보유한 의료기기(울쎄라, 올리지오, 인모드, 슈링크 유니버스, 피코슈어, 포텐자, 소프웨이브, 젠틀맥스 프로)를 2×4 카드 그리드로 나열하며, 각 카드를 클릭하면 해당 장비의 상세 페이지로 이동합니다.
- **상세 페이지** (`html_resource/ulthera.html`, `oligio.html`, `inmode.html`): 세 페이지가 공통 서브 네비게이션(탭)으로 연결되어 있으며, 동일한 섹션 구조를 따릅니다.
  - **페이지 배너:** "· 리프팅 센터 ·" 라벨 + 시술명 + 한 줄 소개 + 보조 설명
  - **서브 네비게이션:** 울쎄라 / 올리지오 / 인모드 탭 (현재 페이지 하이라이트)

### 1.3 공지사항 및 이벤트 (Notice & Events)
- **목록 페이지:** 관리자가 업로드한 썸네일 사진과 제목이 카드형 레이아웃으로 나열됩니다.
- **상세 페이지:** 게시물을 클릭하면 이동하는 단순 페이지 구조입니다.
  - **구성:** 게시물 제목(상단) + 업로드된 고화질 사진(중앙) + 상세 설명 텍스트(하단)
- **관리 로직:** 관리자 페이지에서 제목, 사진, 본문 내용을 입력하면 D1과 R2에 각각 저장되며 Astro의 Dynamic Routing으로 상세 페이지가 자동 생성됩니다.

### 1.4 프로모션 및 예약 관리 (`html_resource/promotion.html`)
- **목록 페이지:** 카테고리 탭(소프웨이브 런칭, 3월 이벤트, 첫방문 혜택, 보톡스/윤곽주사, 리프팅, 필러, 스킨케어)으로 시술을 분류하며, 각 시술은 카드형 리스트로 나열됩니다.
- **구성:** 카테고리 뱃지 + 시술명 + 간단 설명 + 프로모션 가격(정가 대비 할인율 표시) + 체크박스(찜하기)
- **정렬:** 인기순 / 낮은가격순 / 높은가격순 정렬을 지원합니다.
- **예약 플로우:** 사용자가 체크박스로 원하는 시술을 복수 선택한 뒤, 하단 고정 "예약상담하기" 버튼을 탭하면 바텀시트가 올라옵니다.
  - **1단계 (바텀시트):** 선택한 시술 목록과 합계 금액(VAT 포함)을 확인합니다.
  - **2단계 (예약 폼):** "찜한 목록 예약하기" 버튼을 누르면 예약 폼이 나타나며, 상품명(자동 입력) / 희망 예약 일자 / 희망 예약 시간 / 예약 내용(자유 입력) / 이름 / 연락처를 기입 후 예약을 제출합니다.
- **관리 로직:** 관리자 페이지에서 카테고리, 시술명, 설명, 정가, 할인가, 뱃지 텍스트 등을 입력하면 D1에 저장되며, 프로모션 목록 페이지에 실시간 반영됩니다. 예약 신청 데이터는 D1에 저장되고, 관리자 페이지의 예약 관리 탭에서 일자별로 조회·확인할 수 있습니다.

## 2. 회원 체계 (User System)

### 2.1 인증 서비스
- **Clerk** 를 사용하여 회원가입, 로그인, 비밀번호 재설정, 세션 관리를 모두 처리합니다.
- 이름, 이메일, 비밀번호 등 인증 정보는 Clerk에서 관리하며, D1 `users` 테이블에는 `clerk_id`를 키로 앱 고유 데이터(역할, 약관 동의 등)만 저장합니다.

### 2.2 회원 유형
| 유형 | `role` 값 | 설명 |
| :--- | :--- | :--- |
| **일반 회원** | `user` | 웹사이트 이용, 예약 내역 조회 등 |
| **관리자** | `admin` | 관리자 페이지(`/admin/*`) 접근 가능, 콘텐츠/예약/회원 관리 |

- 최초 관리자: `ADMIN_EMAILS` 환경변수에 등록된 이메일로 가입 시 자동 `admin` 부여
- 이후 관리자 추가/해제: 관리자 페이지 내 **회원 관리**(`/admin/members`)에서 기존 관리자가 처리

## 3. 인증 및 계정 관리 (Authentication — Clerk)

### 3.1 회원가입 (`/auth/register`)
| 항목 | 내용 |
| :--- | :--- |
| **UI** | Clerk `<SignUp>` 컴포넌트 사용 (중앙 정렬 카드형) |
| **Clerk 처리** | 이름, 이메일, 비밀번호 입력 및 검증은 Clerk가 자동 처리 |
| **약관 동의** | Clerk `unsafeMetadata`에 약관 동의 상태 저장하거나, 회원가입 완료 후 약관 동의 페이지로 리다이렉트 |
| **D1 연동** | Clerk Webhook(`user.created`) 또는 가입 완료 콜백에서 `users` 테이블에 `clerk_id`, `role`, `agreed_terms`, `agreed_privacy` 삽입 |
| **성공** | 가입 완료 → 홈 페이지로 리다이렉트 |
| **역할** | 가입 시 기본 역할은 `user`. 관리자 역할은 D1에서 직접 변경하거나 기존 관리자가 부여 |

### 3.2 로그인 (`/auth/login`)
| 항목 | 내용 |
| :--- | :--- |
| **UI** | Clerk `<SignIn>` 컴포넌트 사용 (중앙 정렬 카드형) |
| **Clerk 처리** | 이메일+비밀번호 검증, 세션 쿠키 발급 모두 Clerk가 처리 |
| **성공** | 이전 페이지 또는 홈으로 리다이렉트 |
| **실패** | Clerk 자체 에러 메시지 표시 |

### 3.3 비밀번호 재설정
| 항목 | 내용 |
| :--- | :--- |
| **처리** | Clerk `<SignIn>` 컴포넌트 내 "비밀번호 찾기" 링크로 자동 지원 |
| **흐름** | Clerk가 이메일 발송 → 토큰 검증 → 비밀번호 변경까지 모두 자동 처리 |
| **별도 페이지 불필요** | 기존 `/auth/find-password`, `/auth/reset-password` 페이지 제거 |

### 3.4 세션 관리
| 항목 | 내용 |
| :--- | :--- |
| **관리** | Clerk SDK가 세션 쿠키 발급/갱신/무효화 자동 처리 |
| **로그아웃** | Clerk `<UserButton>` 또는 `signOut()` 호출로 처리 |
| **미들웨어** | Astro 미들웨어에서 `clerkMiddleware()` 사용하여 인증 상태 확인 |

### 3.5 마이페이지 (`/mypage`)
| 항목 | 내용 |
| :--- | :--- |
| **접근 조건** | 로그인 필수 (미로그인 시 `/auth/login`으로 리다이렉트) |
| **UI** | 중앙 정렬 카드형 레이아웃 |
| **표시 내용** | `"{name}님, 안녕하세요"` 인사 메시지 (Clerk `user.firstName` / `user.fullName` 사용) |
| **회원탈퇴** | 하단에 "회원탈퇴" 링크 또는 버튼 → 클릭 시 탈퇴 확인 모달 표시 |

### 3.6 회원탈퇴
| 항목 | 내용 |
| :--- | :--- |
| **진입** | 마이페이지 내 "회원탈퇴" 버튼 클릭 |
| **확인 모달** | "정말 탈퇴하시겠습니까? 탈퇴 시 모든 정보가 삭제되며 복구할 수 없습니다." 경고 메시지 |
| **처리 로직** | Clerk Backend API로 사용자 삭제 (`clerkClient.users.deleteUser()`) → D1 `users` 테이블에서 해당 행 삭제 → 세션 무효화 |
| **성공** | 탈퇴 완료 → 홈 페이지(`/`)로 리다이렉트 + "회원 탈퇴가 완료되었습니다" 토스트 메시지 |

### 3.7 인증 미들웨어 (사용자 경로)
| 경로 | 검증 방식 |
| :--- | :--- |
| `/auth/*` | 인증 불필요 (비로그인 접근 가능) |
| `/mypage` | Clerk 미들웨어로 인증 확인 (미로그인 시 `/auth/login` 리다이렉트) |
| `/api/auth/withdraw` | Clerk 인증 필수 |
| `/api/public/*` | 인증 불필요 (프론트엔드 데이터 조회용) |

## 4. 문의 메일 발송

### 4.1 개요
웹사이트 내 문의하기 폼에서 고객이 작성한 내용을 관리자 이메일로 발송합니다.

### 4.2 발송 방식
- **서비스:** Resend API
- **발신:** `noreply@luminus-clinic.com` (또는 Resend에 등록된 도메인)
- **수신:** 관리자 이메일 (`ADMIN_EMAIL` 환경변수)
- **메일 내용:** 고객 이름, 연락처, 문의 내용, 접수 일시

## 5. API 엔드포인트 (사용자)

### 5.1 계정
| Method | Endpoint | 설명 |
| :--- | :--- | :--- |
| POST | `/api/auth/withdraw` | 회원탈퇴 (Clerk 사용자 삭제 + D1 행 삭제) |

> 회원가입, 로그인, 로그아웃, 비밀번호 재설정 등은 Clerk SDK가 자체 처리하므로 별도 API 불필요

### 5.2 Clerk Webhook
| Method | Endpoint | 설명 |
| :--- | :--- | :--- |
| POST | `/api/webhooks/clerk` | Clerk 이벤트 수신 (`user.created` → D1 `users` 행 생성, `user.deleted` → D1 행 삭제) |

### 5.3 문의 메일
| Method | Endpoint | 설명 |
| :--- | :--- | :--- |
| POST | `/api/contact` | 문의 메일 발송 (Resend API로 관리자에게 메일 전달) |
